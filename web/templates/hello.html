Flask(파이썬 플라스크) - (app.py 다 만들고 난 뒤에 한것) ​ 기존 시스템에 영향을 주지 않고 flask서버를 하기 위해 하는 작업 중 하나 ​ vmware(가상머신) 우분투(리눅스)로 해본 예시) ​ ① pip install pipenv --user (pipenv --user를 인스톨) ② pipenv --two (파이썬 2.x버전으로 하겠다는것을 지정) ③ pipenv install flask ④ pip shell(치고 난뒤 pip shell를 치고 접속(별도의 독립된 가상 환경)되면 설치된게 확인됨 그냥 치면 없다고 나옴) ⑤ git clone 주소(github nw주소) 치고 git에 있는걸가져옴(지금까지 학원에서 짠 nw폴더에있는 소스를 클론으로 가져옴) ​ ⑥ run.bat을 run.sh로 변환후 #!/bin/bash export FLASK_APP=app.py flask run --host=0.0.0.0으로 수정후 ⑦ ./run.sh 하면 서버가 켜진다(퍼미션 디나이가 뜰경우 실행권한을 줘야하므로 chmod +x ./run.sh로 권한을 줌) 웹서버를 역할을 해줌 아파치랑 nginx ⑧ sudo apt install nginx(웹서버 설치) ⑨ pipenv install gunicorn(주니콘) 설치후 ⑩ pipenv shell에서 gunicorn app:app -b localhost:5000&(app.py에 app을 실행 -b 바인드 하겠다 로컬호스트 5천번을 &(백 그 라운드)로 실행하겠다) ​ ○​ 백그라운드에서 실행되는지 확인 ○ ps로 실행중인 프로세스 확인 ★ kill -9 실행중인 프로세스pid ★ 를 입력하면 프로세스 강제종료 curl http://localhost:5000을 치면 백그라운드에 실행중인 html이 나옴 ⑪ sudo -i(루트 권한을 얻음) ⑫ vi /etc/nginx/sites-available/default location 부분에 try_files $url $url/ =404; 앞에 #을 붙여 처리하고 아래에 proxy_pass http://127.0.0.1:5000;을 붙여줌으로서 데이터가 들어왔을 때 위주소로 보낸다는 의미(?) - 정확하진않음 ⑬ 위 과정을한 뒤에 service nginx restart(바로 쉘이 떨어져야함 아니면 오탈자확인) ​ 위 과정이 끝나면 내가 작성한 코드가 서버에서 작동하도록 변경 해준 것 nginx와 gunicorn과 flask를 연동해서 수행
